# VK Vote Audit

Инструмент аудита голосований ВКонтакте. Выявляет накрутку через анализ профилей проголосовавших.
Zero dependencies — только Python stdlib.

## Что делает

1. **collect.py** — собирает данные через VK API:
   - Информация об опросе (варианты, голоса)
   - Списки проголосовавших (для неанонимных опросов)
   - Профили всех голосовавших (город, активность, фото, статус)
   - Список участников группы

2. **collect_deep.py** *(опционально)* — глубокий OSINT-анализ:
   - Списки друзей подозрительных аккаунтов → кластеры ботоферм
   - Подписки на группы → выявление общих групп ботов
   - Использует VK `execute` для батчинга (25 API-вызовов за запрос)

3. **report.py** — генерирует HTML-отчёт:
   - Оригинальные vs скорректированные результаты
   - Уровень риска накрутки по вариантам
   - Временная шкала регистрации аккаунтов (оценка по ID)
   - География голосования
   - Полные списки подозрительных аккаунтов
   - Деактивированные/забаненные аккаунты
   - Кластеры друзей среди подозрительных *(OSINT+)*
   - Общие группы подозрительных *(OSINT+)*
   - Итоговый вердикт с методологией

## Требования

- Python 3.10+ (stdlib only, без внешних зависимостей)
- VK API токен с правами `groups`, `stats`, `wall`

## Установка и запуск

```bash
git clone https://github.com/ZombiePm/vk-vote-audit.git
cd vk-vote-audit

# 1. Настроить переменные
cp .env.example .env
# Отредактировать .env — указать VK_TOKEN и параметры опроса

# 2. Собрать данные
python collect.py

# 3. (Опционально) Глубокий OSINT-анализ
python collect_deep.py

# 4. Сгенерировать отчёт
python report.py

# 5. Открыть отчёт
start output/report.html     # Windows
open output/report.html      # macOS
xdg-open output/report.html  # Linux
```

## Получение VK API токена

Через OAuth Implicit Flow:

```
https://oauth.vk.com/authorize?client_id=2685278&scope=groups,stats,wall&redirect_uri=https://oauth.vk.com/blank.html&display=page&response_type=token&v=5.199
```

1. Откройте ссылку в браузере
2. Нажмите «Разрешить»
3. Скопируйте `access_token` из адресной строки
4. Вставьте в `.env`

Токен действует 24 часа.

## Конфигурация (.env)

| Переменная | Описание | Пример |
|---|---|---|
| `VK_TOKEN` | OAuth-токен VK API | `vk1.a.XXXX...` |
| `VK_WALL_POST` | Пост с опросом (ownerID_postID) | `-123456789_123` |
| `VK_GROUP_ID` | ID группы (без минуса) | `123456789` |
| `VK_TARGET_CITY` | Целевой город для гео-анализа | `москва` |

## Методология скоринга

Каждому проголосовавшему присваивается suspicion score:

| Критерий | Баллы |
|---|---|
| Деактивирован/забанен | +3 |
| Неактивен >180 дней | +2 |
| Нет аватарки | +1 |
| Не в группе | +1 |
| Город не указан | +1 |
| Город не совпадает с целевым | +1 |
| Закрытый профиль + нет фото | +1 |
| Нет данных об активности | +1 |
| Аккаунт создан в год опроса+ | +1 |

**Классификация:** 5+ = бот, 4 = подозрительный, 2-3 = слабые признаки, 0-1 = чистый.

Скорректированные результаты = оригинал минус аккаунты с score ≥ 5.

### Глубокий OSINT-анализ (collect_deep.py)

- **Кластеры друзей** — подозрительные аккаунты, дружащие друг с другом, объединяются в кластеры. Кластер с единогласным голосованием — ботоферма.
- **Общие группы** — группы ВК, в которых состоят 3+ подозрительных. Указывает на общий источник ботов.
- **Дата регистрации** — оценка по ID аккаунта (VK выдаёт ID последовательно). Массовое создание аккаунтов в один период — признак ботофермы.

Вдохновлено [vk-osint-ru](https://github.com/OSINT-mindset/vk-osint-ru).
